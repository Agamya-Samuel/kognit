# Dashboard Components Design Plan

## Executive Summary

Create shared dashboard components for `web-admin`, `web-instructor`, and `web-student` apps. The three apps have nearly identical sidebar layouts that are currently duplicated. This plan consolidates them into a single parametric `DashboardShell` component in `packages/shared-components`, adds missing shadcn primitives to `packages/ui`, and establishes a consistent dashboard experience across all roles.

---

## Phase 1: shadcn CLI Setup & New Primitives

### 1.1 Initialize shadcn CLI in `packages/ui`

Create `packages/ui/components.json` to enable `npx shadcn@latest add`:

```json
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "default",
  "rsc": true,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "src/tokens/tailwind.css",
    "baseColor": "slate",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components"
  }
}
```

### 1.2 Add Missing shadcn Components to `packages/ui`

Run `npx shadcn@latest add` for each needed component. These go into `packages/ui/src/components/`:

| Component | Purpose | Used By |
|-----------|---------|---------|
| **Sheet** | Mobile sidebar drawer | DashboardShell (mobile nav) |

### 1.3 Update `packages/ui/src/index.ts`

Export all new components from the barrel file.

## Phase 2: Shared Dashboard Components in `packages/shared-components`

### 2.1 DashboardShell Component

**File:** `packages/shared-components/src/components/dashboard/DashboardShell.tsx`

A single parametric component accepting:

```typescript
interface DashboardShellProps {
  brand: {
    name: string;
    logo?: React.ReactNode;
  };
  navItems: NavItem[];
  user: {
    name: string;
    email?: string;
    avatarUrl?: string;
  };
  headerTitle: string;
  onLogout: () => void;
  children: ReactNode;
}

interface NavItem {
  href: string;
  label: string;
  icon: LucideIcon;
  badge?: string | number;
  children?: NavItem[];
}
```

**Features:**
- Fixed sidebar (w-64) on desktop
- Sheet-based mobile drawer on `< 1024px`
- Sticky top header with breadcrumbs area, user avatar dropdown
- Logout button in sidebar footer
- Active nav item detection via `usePathname()`
- Dark mode support via design tokens

### 2.2 Sub-Components

| Component | File | Description |
|-----------|------|-------------|
| `DashboardSidebar` | `dashboard/DashboardSidebar.tsx` | Desktop sidebar with nav items, brand, logout |

| `MobileSidebar` | `dashboard/MobileSidebar.tsx` | Sheet wrapper for mobile sidebar |
| `NavSection` | `dashboard/NavSection.tsx` | Renders a group of nav items with active state |
| `UserMenu` | `dashboard/UserMenu.tsx` | DropdownMenu with avatar, profile link, logout |

## Phase 3: App Migration

### 3.1 web-admin Migration

**File: `apps/web-admin/src/app/dashboard/layout.tsx`**

Replace inline sidebar layout with:
```tsx
import { DashboardShell } from '@edutech/shared-components';

const navItems = [
  { href: '/dashboard', label: 'Dashboard', icon: LayoutDashboard },
  { href: '/dashboard/users', label: 'Users', icon: Users },
  { href: '/dashboard/instructors', label: 'Instructors', icon: GraduationCap },
  { href: '/dashboard/courses', label: 'Courses', icon: BookOpen },
  { href: '/dashboard/analytics', label: 'Analytics', icon: BarChart3 },
  { href: '/dashboard/settings', label: 'Settings', icon: Settings },
];

<DashboardShell
  brand={{ name: 'EduTech Admin' }}
  navItems={navItems}
  user={{ name: 'Admin' }}
  headerTitle="Admin Dashboard"
  onLogout={() => {}}
>
  {children}
</DashboardShell>
```

**File: `apps/web-admin/src/app/dashboard/page.tsx`**

Replace inline metric cards with shared `MetricCard` / `MetricCardGrid`.

**File: `apps/web-admin/src/components/UserDetailDrawer.tsx`**

Refactor to use `Sheet` from `@edutech/ui` instead of custom drawer.

### 3.2 web-instructor Migration

**File: `apps/web-instructor/src/app/dashboard/layout.tsx`**

Replace inline sidebar layout with `DashboardShell`, passing instructor-specific nav items. Auth guard logic moves to the consuming app.

**File: `apps/web-instructor/src/app/dashboard/page.tsx`**

Replace inline metrics with `MetricCard` / `MetricCardGrid`, activity feed with `ActivityList`, quick actions with `QuickActions`.

**Files: `apps/web-instructor/src/components/EmptyState.tsx`, `ErrorState.tsx`**

## Implementation Order

1. **Phase 1.1-1.3**: Set up shadcn CLI, add components, update exports (~30 min)
2. **Phase 2.1-2.2**: Build DashboardShell + sub-components (~1.5 hr)
3. **Phase 2.3**: Merge EmptyState/ErrorState into shared (~15 min)
4. **Phase 2.4**: Build MetricCard, StatusBadge, ActivityList, QuickActions (~45 min)
5. **Phase 2.5**: Update shared-components barrel exports (~5 min)
6. **Phase 4**: Design token updates (~10 min)
7. **Phase 3.1**: Migrate web-admin (~20 min)
8. **Phase 3.2**: Migrate web-instructor (~20 min)
9. **Phase 3.3**: Migrate web-student + create new dashboard page (~45 min)
