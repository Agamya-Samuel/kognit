# Plan: Design Team Brief — Page-by-Page Screen Map

## Goal
Create a single comprehensive design brief document at `docs/design/01-design-brief.md` that provides a page-by-page screen map for the design team. The document covers all apps: Landing Page, Student, Instructor, and Admin.

## Format
- Page-by-page screen map (per user's answer)
- Mobile + Desktop responsive specs
- Design system NOT dictated (designers define colors/typography/spacing)
- Brand identity starting from scratch
- Only UI/UX-relevant details — no backend code, no DB internals, no deployment details

## Document Structure

### 1. Introduction
- Platform overview (who it's for, what it does)
- Target audience (18-26 college students in India, mobile-first)
- Platform zones overview (4 separate web apps)

### 2. Shared/Auth Pages (common patterns across all apps)
Pages:
1. **Login** — email/password form, OAuth buttons (Google, GitHub), forgot password link, error states (invalid credentials, account locked, too many attempts)
2. **Email Verification** — enter 6-digit code screen, resend code, success confirmation
3. **Registration (Set Password + Name)** — after email verified, password requirements, name field
4. **Forgot Password** — enter email, confirmation sent
5. **Reset Password** — new password entry, confirm password, success → redirect to login
6. **OAuth Redirect** — loading spinner while OAuth processes

### 3. Landing Page App (`eduplatform.com`)
Pages:
1. **Hero Section** — headline, subheading, CTA button ("Start Learning"), background visual
2. **Features Section** — course categories, platform benefits
3. **Social Proof Section** — testimonials, stats
4. **CTA Section** — final call-to-action before footer
5. **Footer** — nav links, social media, copyright

### 4. Student App (`student.eduplatform.com`)
Pages:
1. **Dashboard/Home** — enrolled courses, upcoming live classes, progress summary, recent activity
2. **Course Catalog** — browse/search courses, filters (domain, price, rating), course cards
3. **Course Detail** — title, description, syllabus (sections + lectures), instructor bio, reviews, price, enroll CTA, free preview player
4. **Video Player Page** — HLS video player, progress tracking, playback controls (speed, quality), lecture sidebar, completion tracking
5. **Live Class Page** — LiveKit video, chat panel, raise hand, attendance, live indicator
6. **My Courses** — enrolled courses grid, progress bars, continue learning CTAs
7. **Assignment List** — per-course assignment list, submission status badges
8. **Assignment Submission** — assignment details, file upload area, code editor (for code submissions), submit button, submission history
9. **Quiz Page** — questions (MCQ/short answer), timer, progress indicator, submit & auto-grade
10. **Community/Forums** — channel list (course channels + general), channel view with messages, reply threads, flag/report
11. **Certificates** — earned certificates list, preview card, download PDF, share (LinkedIn), verification link
12. **Certificate Verification** — public page, certificate details, validity confirmation
13. **Placement Profile Builder** — skills, resume upload, projects, placement status
14. **Job Board** — curated job listings, filters by domain, job detail cards
15. **Notifications Center** — notification list, read/unread, filter by type, mark all read
16. **Checkout/Payment** — order summary, Razorpay integration, payment processing state
17. **Payment Success** — confirmation, receipt details, start learning CTA
18. **Profile/Settings** — avatar, name, email, password change, notification preferences

### 5. Instructor App (`instructor.eduplatform.com`)
Pages:
1. **Instructor Application Form** (pre-approval) — bio, expertise, social links, documents upload, submission confirmation
2. **Application Status** (pre-approval) — pending/review status, timeline, email notification setup
3. **Dashboard** — enrollment count, revenue summary, upcoming classes, recent student activity
4. **Course List** — published/draft courses, create new course CTA, quick stats per course
5. **Course Creation Wizard** — multi-step: (1) Basic info (title, description, thumbnail, domain, pricing), (2) Sections & lectures builder, (3) Content upload, (4) Review & publish
6. **Course Editor** — section/lecture list, drag-and-drop reorder, add/edit/delete lectures, set free preview toggle
7. **Content Upload** — drag-and-drop file upload, progress bar, transcoding status, retry on failure
8. **Live Class Scheduler** — date/time picker, duration, attach to course, notify students toggle
9. **Live Class Room** — LiveKit host view, participant management, recording controls, start/end class
10. **Student Management** — enrolled students table, search/filter, individual student progress view
11. **Student Progress Detail** — per-student: lecture completion, assignment submissions, quiz scores, time spent
12. **Assignment Grading** — submission list, view submission, grade input, feedback textarea, grade history
13. **Analytics Dashboard** — enrollment trends, watch completion rates, drop-off points, revenue charts, time period filters
14. **Revenue Page** — total earnings, per-course breakdown, payout history, upcoming payout schedule, revenue share display
15. **Banking/Payout Setup** — bank details form, verification status, Razorpay seller account linking
16. **Profile/Settings** — avatar, bio, expertise, social links, notification preferences
17. **Notifications Center** — same pattern as student

### 6. Admin App (`admin.eduplatform.com`)
Pages:
1. **Dashboard/Overview** — platform stats (total users, courses, revenue), system health, recent activity, pending moderation count
2. **User Management** — users table (students + instructors), search/filter by role/status, user detail view, actions (activate/deactivate, role change, password reset assist)
3. **User Detail** — profile info, activity timeline, action history, audit log for user
4. **Instructor Approval Queue** — pending applications, review details, approve/reject with reason, email notification
5. **Course Moderation Queue** — flagged courses, review details, approve/reject/suspend, moderation history
6. **Content Review** — course preview, lecture list, flag history, moderation actions
7. **Platform Analytics** — revenue overview, enrollment trends, user growth, engagement metrics, date range filters, chart controls
8. **Revenue Analytics** — platform revenue, instructor payouts, transaction history, Razorpay sync status
9. **Report Generation** — report type selector (revenue, enrollments, user activity), date range, format (CSV/PDF), generation progress, download
10. **Institution Management** (Phase 2) — institution list, account details, seat management
11. **Support Tickets** (Phase 2) — ticket queue, status filters, ticket detail, response thread
12. **Audit Log** — action log table, filter by user/action/entity/date, export
13. **Security Monitoring** — security events, active threats, access patterns
14. **System Settings** — platform configuration, feature toggles, maintenance scheduling
15. **Notifications Center** — admin notification list

### 7. Phase 2 Pages (Future Scope)
- Institution Dashboard (seat management, cohort reports)
- Support Tickets (ticket queue, response thread)
- Instructor Bulk Upload (batch upload, file-by-file status)
- Advanced Community Features (reactions, moderation tools)
- Mock Interviews & Referrals (placement phase 2)
- Multi-language Support consideration

### 8. Cross-Platform UX Patterns
- Navigation patterns (sidebar, top nav, bottom nav for mobile)
- Toast notification patterns
- Loading skeleton patterns
- Empty state patterns
- Error state patterns
- Confirmation dialog patterns
- Pagination/infinite scroll patterns

For each page, the document will include:
- **Purpose**: What the page does
- **Entry Points**: How users get here
- **Key UI Elements**: Layout description and components
- **Data Displayed**: What information is shown
- **User Actions**: What the user can do
- **States**: Empty, Loading, Error, Success states
- **Mobile Considerations**: Responsive behavior, mobile-specific patterns
- **Exit Points**: Where users go from here
